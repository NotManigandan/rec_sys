## 2. `fedsys/config.py`

Central home for every runtime knob. Both the coordinator and all nodes import from here, so there is exactly one place to change any setting.

---

### `CoordinatorConfig` (dataclass)

Holds every parameter the coordinator server needs.

| Field | Type | Default | Meaning |
|---|---|---|---|
| `host` | str | `"0.0.0.0"` | Network interface to bind (all interfaces by default) |
| `port` | int | `50051` | TCP port for the gRPC server |
| `min_nodes` | int | `2` | **K** ŌĆö minimum updates required before FedAvg runs |
| `total_nodes` | int | `2` | **N** ŌĆö total expected nodes; used to assign data partitions |
| `round_timeout_seconds` | float | `120.0` | How long to wait before dropping stragglers |
| `num_rounds` | int | `10` | Total FL rounds to run before the coordinator exits |
| `chunk_size_bytes` | int | `4 MB` | Size of each streaming chunk sent over gRPC |
| `log_dir` | str | `"logs"` | Directory where telemetry files are written |
| `db_path` | str | `"logs/telemetry.db"` | SQLite database path |
| `log_file` | str | `"logs/telemetry.jsonl"` | JSONL output path |
| `grpc_options` | list | see below | gRPC channel options passed to `grpc.server()` |

`grpc_options` always includes:
- `max_send_message_length = -1` (unlimited)
- `max_receive_message_length = -1` (unlimited)
- Keepalive settings to detect dead connections

---

### `NodeConfig` (dataclass)

Holds every parameter a training node needs.

| Field | Type | Default | Meaning |
|---|---|---|---|
| `node_id` | str | random 8-char UUID | Unique identifier for this node |
| `coordinator_host` | str | `"localhost"` | Where to find the coordinator |
| `coordinator_port` | int | `50051` | Coordinator's port |
| `local_epochs` | int | `5` | SGD passes over local data per FL round |
| `batch_size` | int | `256` | Mini-batch size for the local training loop |
| `learning_rate` | float | `1e-3` | Adam learning rate |
| `device` | str | `"cpu"` | `"cpu"`, `"cuda"`, or `"cuda:0"` etc. |
| `data_partition` | int | `0` | Set by coordinator after registration |
| `num_partitions` | int | `3` | Total shards; set by coordinator |
| `chunk_size_bytes` | int | `4 MB` | Chunk size for streaming model updates |
| `log_dir` | str | `"logs"` | Telemetry directory |
| `db_path` | str | node-specific `.db` path | Per-node SQLite file |
| `log_file` | str | node-specific `.jsonl` path | Per-node JSONL file |
| `grpc_options` | list | same as coordinator | gRPC channel options |

**`coordinator_address` (property)** ŌĆö Returns `"host:port"` as a single string. Used when constructing the gRPC channel.

---

### `ModelConfig` (dataclass)

Defines the model architecture. Both coordinator and nodes must use identical values or `load_state_dict` will crash.

| Field | Type | Default | Meaning |
|---|---|---|---|
| `model_type` | str | `"simple"` | `"simple"` ŌåÆ `TwoLayerModel`; `"ncf"` ŌåÆ `NCFRecommender` |
| `num_users` | int | `1_000` | Size of the user embedding table |
| `num_items` | int | `500` | Size of the item embedding table |
| `embedding_dim` | int | `16` | Dimension of both user and item embeddings |
| `mlp_hidden` | list[int] | `[64]` | Hidden layer widths. For `TwoLayerModel` only the first entry is used. For `NCFRecommender` each entry becomes one hidden layer. |

---

### `large_model_config()` ŌåÆ `ModelConfig`

A convenience function that returns a pre-filled `ModelConfig` for the production ~300 M parameter NCF model:
- `model_type = "ncf"`
- `num_users = 1_000_000`
- `num_items = 500_000`
- `embedding_dim = 128`
- `mlp_hidden = [256, 4096, 8192, 8192, 4096, 2048, 1024]`

Call this in the run scripts instead of spelling out all the values manually.

---

### `ensure_log_dir(cfg)` ŌåÆ None

Creates `cfg.log_dir` on disk if it doesn't already exist. Called at the start of `serve()` and `FederatedNode.__init__()` so telemetry files always have a directory to land in.

## 3. `fedsys/privacy.py`

A byte-level middleware layer that sits between the gRPC transport and the model bytes. Currently both functions are identity (pass-through), but their positions in the call stack are fixed so future DP or encryption logic slots in here without touching any networking or training code.

---

### `apply_privacy_transform(payload, *, context=None)` ŌåÆ `bytes`

Called by the **sender** immediately before chunking a payload for transmission.

- **`payload`**: The raw `bytes` from `torch.save(state_dict, BytesIO)`.
- **`context`**: Optional string hint such as `"node-0->coordinator"` or `"broadcast->node-1"`. Useful for routing different transforms per direction.
- **Returns**: Transformed bytes. Currently returns `payload` unchanged.
- **Where it's called**:
  - `coordinator/server.py ŌåÆ FetchGlobalModel()` before splitting into chunks (outbound broadcast)
  - `node/client.py ŌåÆ _send_update()` before splitting into chunks (outbound update)

**Future implementation stub** shown in comments:
1. Deserialize bytes back to a tensor dict.
2. Clip gradients by L2 norm.
3. Add Gaussian noise (DP mechanism).
4. Re-serialize and return.

---

### `reverse_privacy_transform(payload, *, context=None)` ŌåÆ `bytes`

Called by the **receiver** immediately after reassembling all chunks.

- **`payload`**: Reassembled full byte string.
- **`context`**: Same hint string as above.
- **Returns**: Un-transformed bytes. For one-way mechanisms (DP noise addition), this is also identity. For encryption, this would decrypt.
- **Where it's called**:
  - `coordinator/server.py ŌåÆ SendLocalUpdate()` after chunk reassembly (inbound update)
  - `node/client.py ŌåÆ _fetch_global_model()` after chunk reassembly (inbound model)

---

### `_serialize(tensor_dict)` ŌåÆ `bytes` *(private)*

Stub helper. Saves a `{str: torch.Tensor}` dict to a `BytesIO` buffer using `torch.save` and returns the raw bytes. Exists so future transform code doesn't repeat serialization boilerplate.

---

### `_deserialize(payload)` ŌåÆ `dict` *(private)*

Inverse of `_serialize`. Loads bytes back into a tensor dict mapped to CPU. Provided so future DP code can operate at the tensor level.

---

## 4. `fedsys/logging_async.py`

The telemetry backbone of the entire framework. Every timing measurement, payload size, state transition, and hardware metric flows through this module. The key design constraint: **logging must never block the thread doing training or networking**.

---

### Module-level constant: `_POISON`

A sentinel Python object (not a string, not `None`). When placed on the queue it signals the drain thread to flush everything and exit. Using a unique object prevents accidental collision with any real log record.

---

### `AsyncTelemetryLogger`

A thread-safe, non-blocking logger that writes to JSONL and SQLite simultaneously via a background thread.

#### `__init__(log_file, db_path, maxsize=10_000, flush_interval=1.0)`

Sets up the logger's internal state. **Does not start the background thread** ŌĆö call `.start()` first.

- `log_file`: Path for the append-only JSONL file. One JSON object per line.
- `db_path`: Path for the SQLite database file.
- `maxsize`: Maximum records allowed in the in-memory queue at once. If the drain thread falls behind and the queue fills up, new records are silently dropped (counted in `self._dropped`) rather than blocking the caller.
- `flush_interval`: How often (seconds) the drain thread commits accumulated SQLite inserts even if the queue still has items. Batching writes is much faster than committing after every single row.

---

#### `start()` ŌåÆ `self`

Creates `log_dir` and `db_dir` on disk (via `os.makedirs`), then launches the `_drain_loop` as a **daemon thread** named `"telemetry-drain"`. Returns `self` so you can chain: `logger = AsyncTelemetryLogger(...).start()`.

The thread is a daemon, meaning Python will not wait for it on normal program exit ŌĆö but `stop()` should always be called for a graceful flush.

---

#### `log(record: dict)` ŌåÆ `None`

The hot path called from training loops and gRPC handlers. **Never blocks.**

1. If `"ts"` is not in the record, prepends a Unix timestamp via `time.time()`.
2. Attempts `queue.put_nowait(record)` ŌĆö returns immediately.
3. If the queue is full, increments `self._dropped` under a lock and discards the record.

---

#### `stop(timeout=10.0)` ŌåÆ `None`

Graceful shutdown:
1. Puts the `_POISON` sentinel on the queue.
2. Calls `thread.join(timeout)` to wait for the drain thread to flush and close files.

Always call this at the end of a coordinator or node session.

---

#### `_drain_loop()` *(private, runs in background thread)*

The actual I/O worker. Runs until it sees `_POISON`.

**Setup:**
- Opens the JSONL file in append + line-buffered mode. Line buffering means each `write()` call flushes to the OS immediately, giving low JSONL latency without an explicit `flush()` on every record.
- Opens (or creates) the SQLite connection and calls `_init_db()`.

**Main loop:**
1. Calls `queue.get(timeout=flush_interval)`.
2. If the queue times out (no items), calls `flush()` immediately to commit any pending SQLite rows.
3. If the item is `_POISON`, calls `flush()` and breaks.
4. Otherwise: writes the record to JSONL immediately, appends it to the `pending` list.
5. If `flush_interval` seconds have passed since the last commit, calls `flush()`.

**`flush()` inner function:**
- Uses `executemany()` to batch-insert all pending records into the `events` table in one transaction.
- Extracts `ts`, `event`, `node_id`, `epoch` as top-level columns for fast indexed lookups.
- Stores the entire JSON record in the `data` column for complete access.
- Catches and prints SQLite errors without re-raising, so a disk error never crashes the training process.

---

#### `_init_db(conn)` *(static, private)*

Creates the `events` table and two indices if they don't exist:

```sql
CREATE TABLE IF NOT EXISTS events (
    id      INTEGER PRIMARY KEY AUTOINCREMENT,
    ts      REAL    NOT NULL,     -- Unix timestamp
    event   TEXT    NOT NULL,     -- event name string
    node_id TEXT,                 -- which node emitted this
    epoch   INTEGER,              -- FL round number
    data    TEXT                  -- full JSON blob
);
CREATE INDEX IF NOT EXISTS idx_event ON events(event);
CREATE INDEX IF NOT EXISTS idx_epoch ON events(epoch);
```

The two indices make queries like `WHERE event='LOCAL_TRAINING_END'` and `WHERE epoch=3` fast.

---

### `TimedBlock`

A context manager that wraps a code block, measures its wall-clock duration, and emits two telemetry events (`_START` and `_END`) without blocking.

#### `__init__(logger, event, **extra)`

- `logger`: The `AsyncTelemetryLogger` to post events to.
- `event`: Base name, e.g. `"H2D_TRANSFER"`. Will produce `H2D_TRANSFER_START` and `H2D_TRANSFER_END`.
- `**extra`: Any additional key-value pairs to include in both events (e.g. `node_id="n0"`, `epoch=2`).

#### `__enter__()` ŌåÆ `self`

Records `time.perf_counter()` as `self._t0` and logs `{event}_START`.

#### `__exit__(*_)` ŌåÆ `None`

Computes `elapsed_ms = (perf_counter() - t0) * 1000` and logs `{event}_END` with `elapsed_ms` included.

## 5. `fedsys/interceptors.py`

gRPC interceptors that wrap every RPC call without modifying any business logic. Both interceptors push records to an `AsyncTelemetryLogger` non-blocking queue.

---

### Module helpers

#### `_byte_size(obj)` ŌåÆ `int` *(private)*

Calls `obj.ByteSize()` on a protobuf message to get its serialized byte count. Returns `0` on any exception so interceptor errors are never fatal.

#### `_now_ms()` ŌåÆ `float` *(private)*

Returns current time in milliseconds using `time.perf_counter()`. Used for precise RTT and handler latency measurements.

---

### `_MeasuredRequestIterator` *(private class)*

Wraps a client-side streaming **outbound** iterator. Every time the caller pulls the next chunk via `__next__()`, the wrapper:
1. Forwards the call to get the real chunk from the underlying iterator.
2. Calls `_byte_size(msg)` and adds to `self._total_bytes`.
3. Increments `self._chunks`.
4. Logs a `STREAM_CHUNK_SENT` event with the chunk byte size.
5. Returns the chunk to the caller.

Used inside `TelemetryClientInterceptor.intercept_stream_unary()` to observe every chunk of a `SendLocalUpdate` call without buffering the entire payload.

---

### `_MeasuredResponseIterator` *(private class)*

Wraps a server-side streaming **inbound** iterator (e.g. `FetchGlobalModel` response). Every `__next__()` call:
1. Tries to get the next chunk. If `StopIteration` is raised (stream done), logs `STREAM_RECV_COMPLETE` with total bytes, chunk count, and RTT (time since first byte sent), then re-raises to properly terminate the iteration.
2. Otherwise logs `STREAM_CHUNK_RECV` with the chunk size.

Used inside `TelemetryClientInterceptor.intercept_unary_stream()`.

---

### `TelemetryServerInterceptor`

Implements `grpc.ServerInterceptor`. Wraps every incoming RPC handler to capture the server-side view of each call.

#### `__init__(logger, node_id="coordinator")`

Stores the logger and a node identifier (always `"coordinator"` for the server side).

---

#### `_wrap_unary_unary(fn, method)` ŌåÆ callable *(private)*

Returns a wrapper for simple requestŌåÆresponse RPCs (used for `Register`).

The wrapper:
1. Measures request byte size.
2. Logs `RPC_RECV` with method name and request size.
3. Calls the real handler `fn(request, context)`.
4. Logs `RPC_HANDLER_END` with response size and elapsed ms.

---

#### `_wrap_unary_stream(fn, method)` ŌåÆ callable *(private)*

Returns a wrapper for server-streaming RPCs (used for `FetchGlobalModel`).

The wrapper:
1. Logs `RPC_RECV` on the incoming request.
2. Iterates the real generator `fn(request, context)`, counting bytes and chunks per yielded item.
3. After the generator is exhausted, logs `RPC_HANDLER_END` with total bytes sent and chunk count.

This is implemented as a generator itself (`yield chunk` inside) so gRPC can stream chunks to the client as they are produced.

---

#### `_wrap_stream_unary(fn, method)` ŌåÆ callable *(private)*

Returns a wrapper for client-streaming RPCs (used for `SendLocalUpdate`).

The wrapper:
1. Defines an inner generator `measured_iter()` that yields each chunk from the real `request_iterator` while logging `STREAM_CHUNK_RECV` per chunk.
2. Passes `measured_iter()` to the real handler instead of the raw iterator.
3. After the handler returns, logs `RPC_HANDLER_END` with total bytes received.

---

#### `intercept_service(continuation, handler_call_details)` ŌåÆ `RpcMethodHandler`

The required entry point for `grpc.ServerInterceptor`. Called once per incoming RPC before the handler runs.

1. Calls `continuation(handler_call_details)` to get the real handler.
2. Inspects which slot is populated (`unary_unary`, `unary_stream`, or `stream_unary`).
3. Replaces that slot with the appropriate wrapper from above.
4. Returns the modified handler object.

Stream-stream RPCs are passed through unchanged (not used in this proto).

---

### `TelemetryClientInterceptor`

Inherits from all four gRPC client interceptor base classes:
- `grpc.UnaryUnaryClientInterceptor` ŌåÆ for `Register`
- `grpc.UnaryStreamClientInterceptor` ŌåÆ for `FetchGlobalModel`
- `grpc.StreamUnaryClientInterceptor` ŌåÆ for `SendLocalUpdate`
- `grpc.StreamStreamClientInterceptor` ŌåÆ pass-through

#### `__init__(logger, node_id)`

Stores the logger and the node's ID string (used as context in log records).

#### `_meta(details)` ŌåÆ `dict` *(private)*

Returns `{"method": details.method, "node_id": self._node_id}`. Every event emitted by this interceptor includes these two fields.

---

#### `intercept_unary_unary(continuation, client_call_details, request)`

Intercepts the `Register` RPC.
1. Logs `CLIENT_RPC_SEND` with outbound byte size.
2. Calls `continuation(...)` to actually send the RPC.
3. Tries to call `resp.result()` on the future to get the response size.
4. Logs `CLIENT_RPC_RECV` with response size and RTT.

---

#### `intercept_unary_stream(continuation, client_call_details, request)`

Intercepts `FetchGlobalModel` (node sends one request, server streams back many chunks).
1. Logs `CLIENT_RPC_SEND` with request size and records start time `t0`.
2. Calls `continuation(...)` to start the stream.
3. Wraps the resulting iterator in `_MeasuredResponseIterator`, which logs each incoming chunk and the final RTT when the stream ends.

---

#### `intercept_stream_unary(continuation, client_call_details, request_iterator)`

Intercepts `SendLocalUpdate` (node streams many chunks, server replies once).
1. Logs `CLIENT_STREAM_START` and records `t0`.
2. Wraps `request_iterator` in `_MeasuredRequestIterator` so each outbound chunk is logged.
3. Passes the wrapper to `continuation(...)`.
4. After the call returns, logs `CLIENT_STREAM_END` with total bytes sent, chunk count, and RTT.

---

#### `intercept_stream_stream(...)`

Not used by this proto. Simply calls `continuation(...)` and returns the result unchanged.

## 6. `fedsys/data/synthetic_dataset.py`

Loads pre-generated synthetic training data from CSV files. Has **zero dependency on the Amazon dataset module**.

---

### `SyntheticCSVDataset`

A `torch.utils.data.Dataset` that reads one partition CSV produced by `scripts/generate_synthetic_data.py`.

#### `__init__(data_dir, partition_index=0)`

1. Reads `data_dir/meta.json` and stores `num_users`, `num_items`, `num_partitions`.
2. Opens `data_dir/partition_{partition_index}.csv`.
3. Parses every row into a `(user_id: int, item_id: int, label: float)` tuple.
4. Prints how many samples were loaded.

Raises `FileNotFoundError` with a helpful message if either `meta.json` or the CSV is missing, pointing to the generation script.

---

#### `__len__()` ŌåÆ `int`

## 8. `fedsys/models/recommendation.py`

Two model implementations sharing the same `(user_ids, item_ids) ŌåÆ logits` signature, plus a factory function. The rest of the codebase only calls `build_model(cfg)` ŌĆö it never imports the model classes directly.

---

### `TwoLayerModel`

**The default test model.** ~26 K parameters. Trains in milliseconds on CPU.

Architecture:
```
user_ids  ŌåÆ  Embedding(num_users, embedding_dim)  ŌöĆŌöĆŌöÉ
                                                      cat ŌåÆ Linear(2D, hidden) ŌåÆ ReLU ŌåÆ Linear(hidden, 1)
item_ids  ŌåÆ  Embedding(num_items, embedding_dim)  ŌöĆŌöĆŌöś
```

#### `__init__(cfg: ModelConfig)`

- Creates `nn.Embedding(num_users, embedding_dim)` for users.
- Creates `nn.Embedding(num_items, embedding_dim)` for items.
- `hidden = cfg.mlp_hidden[0]` ŌĆö only the first width value is used.
- `fc1 = nn.Linear(2 * embedding_dim, hidden)` ŌĆö first fully connected layer.
- `fc2 = nn.Linear(hidden, 1)` ŌĆö output layer (logit).
- Weight initialisation: embeddings with `N(0, 0.01)`, `fc1` with Kaiming normal (good for ReLU), `fc2` with Xavier uniform.

#### `forward(user_ids, item_ids)` ŌåÆ Tensor `(B, 1)`

1. Look up `user_embedding(user_ids)` ŌåÆ shape `(B, D)`.
2. Look up `item_embedding(item_ids)` ŌåÆ shape `(B, D)`.
3. Concatenate on last dim ŌåÆ shape `(B, 2D)`.
4. Apply `ReLU(fc1(x))` then `fc2(...)` ŌåÆ shape `(B, 1)`.

Returns raw logits. Apply `sigmoid` for probability, or pass directly to `BCEWithLogitsLoss`.

#### `param_count` (property) ŌåÆ `int`

Total number of trainable parameters.

#### `param_count_str()` ŌåÆ `str`

Human-readable parameter count (e.g. `"26.2 K"`, `"337.9 M"`).

---

### `NCFRecommender`

**The production model.** Default config targets ~300 M parameters for large-scale FL experiments.

Architecture:
```
user_ids  ŌåÆ  Embedding(1M, 128)  ŌöĆŌöĆŌöÉ
                                    cat ŌåÆ [LinearŌåÆLayerNormŌåÆGELU] ├Ś 7 ŌåÆ Linear(1024, 1)
item_ids  ŌåÆ  Embedding(500K, 128) ŌöĆŌöĆŌöś
```

The MLP backbone is built dynamically from `cfg.mlp_hidden`. Each hidden width produces three layers: `Linear`, `LayerNorm`, `GELU`.

#### `__init__(cfg: ModelConfig)`

- Creates user + item embedding tables.
- Loops over `cfg.mlp_hidden`, building `[Linear(in_dim, h), LayerNorm(h), GELU()]` for each width `h`.
- Appends a final `Linear(last_hidden, 1)`.
- Wraps the whole sequence in `nn.Sequential` stored as `self.mlp`.
- Calls `_init_weights()`.

#### `forward(user_ids, item_ids)` ŌåÆ Tensor `(B, 1)`

Same as `TwoLayerModel.forward` but runs through the deep `self.mlp` instead of two manual FC layers.

---

### `build_model(cfg: ModelConfig)` ŌåÆ `nn.Module`

The single factory used throughout the codebase.

```python
if cfg.model_type == "simple":
    return TwoLayerModel(cfg)
if cfg.model_type == "ncf":
    return NCFRecommender(cfg)
raise ValueError(...)
```

Called in `coordinator/server.py` and `node/client.py`. Changing `cfg.model_type` is the only thing needed to switch between architectures.

---

### `_fmt_params(n)` ŌåÆ `str` *(private)*

Formats a raw integer parameter count into a human-readable string:
- `Ōēź 1 B` ŌåÆ `"X.XX B"`
- `Ōēź 1 M` ŌåÆ `"X.X M"`
- `Ōēź 1 K` ŌåÆ `"X.X K"`
- Otherwise ŌåÆ plain integer string

## 9. `fedsys/coordinator/registry.py`

Thread-safe in-memory store. All gRPC servicer threads and the aggregation thread share one instance of `NodeRegistry`. Concurrency is handled by a single reentrant lock (`threading.RLock`) that also backs a `threading.Condition` for efficient blocking.

---

### `NodeRecord` (dataclass)

Plain data container stored per registered node:

| Field | Type | Meaning |
|---|---|---|
| `node_id` | str | Unique identifier supplied by the node |
| `address` | str | Network address the node claims (informational) |
| `local_dataset_size` | int | Number of local samples the node reported |
| `partition` | int | Data shard assigned at registration |
| `registered_at` | float | Unix timestamp of first registration |
| `metadata` | dict | Free-form key-value pairs (e.g. `{"device": "cuda"}`) |

---

### `NodeRegistry`

#### `__init__(total_nodes, min_nodes, round_timeout=120.0)`

Sets up the three internal dicts:
- `_nodes`: `{node_id ŌåÆ NodeRecord}` ŌĆö all registered nodes
- `_pending_updates`: `{node_id ŌåÆ bytes}` ŌĆö gradient payloads for the current epoch
- `_pending_samples`: `{node_id ŌåÆ int}` ŌĆö sample counts for weighted averaging

Also sets `_current_epoch = 0` and `_partition_counter = 0`.

---

#### `register(node_id, address, local_dataset_size, metadata)` ŌåÆ `NodeRecord`

Thread-safe registration. Acquires `self._lock`.

- If `node_id` is already known (node crashed and restarted), updates the existing record in-place and returns it. This prevents duplicate partition assignments.
- If new: assigns `partition = _partition_counter % total_nodes`, increments counter, creates and stores a new `NodeRecord`.

---

#### `active_node_ids()` ŌåÆ `list[str]`

Returns a snapshot list of all registered node IDs under the lock.

#### `get_record(node_id)` ŌåÆ `NodeRecord | None`

Returns the `NodeRecord` for a given ID, or `None`.

#### `registered_count` (property) ŌåÆ `int`

Number of currently registered nodes.

---

#### `submit_update(node_id, epoch, payload_bytes, num_samples)` ŌåÆ `bool`

Called from the gRPC servicer thread when a node's `SendLocalUpdate` stream finishes.

1. Acquires `_update_condition` (which is backed by `_lock`).
2. Rejects if `epoch != self._current_epoch` (stale update from a slow node ŌĆö the round already moved on).
3. Rejects if `node_id` not in `_nodes` (unknown node).
4. Stores payload and sample count in the pending dicts.
5. Calls `notify_all()` on the condition to wake up `wait_for_updates()` if it's sleeping.
6. Returns `True` on acceptance, `False` on rejection.

---

#### `wait_for_updates()` ŌåÆ `(dict[str, bytes], dict[str, int])`

Blocking call used by the aggregation thread. Sleeps until either:
- At least `min_nodes` updates have arrived, OR
- The `round_timeout` deadline passes.

Implementation:
1. Computes `deadline = monotonic() + round_timeout`.
2. Enters `_update_condition` context (holds the lock).
3. Loops: if fewer than `min_nodes` updates, calls `condition.wait(timeout=min(remaining, 5.0))`. The 5-second slice prevents indefinite sleeping if the condition was notified before `wait()` was entered.
4. Exits when K updates are present or time runs out.
5. Copies both pending dicts and returns them.

---

#### `advance_epoch()` ŌåÆ `int`

Called by the aggregation loop after FedAvg completes. Clears `_pending_updates` and `_pending_samples`, increments `_current_epoch`, returns new epoch number.

#### `current_epoch` (property) ŌåÆ `int`

Thread-safe read of the current epoch number.

#### `pending_update_count()` ŌåÆ `int`

Number of updates received for the current epoch (used for logging).

---

## 10. `fedsys/coordinator/aggregator.py`

Performs Federated Averaging (FedAvg) on a set of serialized model state dicts. No gRPC, no privacy ŌĆö pure tensor math.

---

### `StateDict` (type alias)

`Dict[str, torch.Tensor]` ŌĆö the standard PyTorch model state dict type.

---

### `serialize_state_dict(state_dict)` ŌåÆ `(bytes, float)`

Converts a state dict to bytes for transmission.

1. Copies all tensors to CPU: `{k: v.cpu() for k, v in state_dict.items()}`. This is device-agnostic ŌĆö the receiver maps tensors to whatever device it needs.
2. Calls `torch.save(cpu_state, BytesIO())`.
3. Returns `(raw_bytes, elapsed_ms)`. The elapsed time is logged for serde overhead analysis.

---

### `deserialize_state_dict(payload, device="cpu")` ŌåÆ `(StateDict, float)`

Inverse of `serialize_state_dict`.

1. Wraps `payload` bytes in a `BytesIO` object.
2. Calls `torch.load(buf, map_location=device, weights_only=True)`. `weights_only=True` is a security measure that prevents executing arbitrary Python code embedded in the file.
3. Returns `(state_dict, elapsed_ms)`.

---

### `FedAvgAggregator`

#### `__init__(logger, device="cpu")`

Stores the async logger and the target device for aggregation (always CPU to avoid VRAM pressure during aggregation).

---

#### `aggregate(updates, sample_counts, epoch)` ŌåÆ `(StateDict, bytes)`

The main FedAvg implementation.

**Parameters:**
- `updates`: `{node_id: serialized_bytes}` ŌĆö the raw payload from each contributing node
- `sample_counts`: `{node_id: int}` ŌĆö local dataset sizes for weighted averaging
- `epoch`: current FL round number (for log records)

**Algorithm:**
1. Computes `total_samples = sum(sample_counts.values())`.
2. For each node's payload:
   - Deserializes the bytes back to a state dict.
   - Computes `weight = n_samples / total_samples`.
   - If this is the first node: `aggregated = {k: v.float() * weight}`.
   - Otherwise: `aggregated[k] += v.float() * weight` for each parameter.
3. Serializes the aggregated result back to bytes.
4. Returns `(aggregated_state_dict, serialized_bytes)`.

**Telemetry emitted:** `AGGREGATION_START`, `AGGREGATION_DESER` (per node), `AGGREGATION_END` with wall-time, serde time breakdown, and VRAM peak delta.

---

### `_peak_vram_mb()` ŌåÆ `float` *(private)*

Calls `torch.cuda.max_memory_allocated()` divided by 1024┬▓ to get MiB. Returns `0.0` if CUDA is not available or on any exception. Called before and after `aggregate()` to compute the VRAM delta.

---

## 11. `fedsys/coordinator/server.py`

The coordinator's top-level module. Contains the gRPC servicer class, the aggregation background thread function, and the `serve()` entry point.

---

### `FederatedLearningServicer`

Implements the three RPCs defined in `federated.proto`. Each method is called by gRPC on a thread from the thread pool. Long-running blocking work (waiting for K updates, FedAvg) happens in a separate `aggregation-loop` thread, not here.

---

#### `__init__(cfg, model_cfg, registry, aggregator, logger)`

Sets up:
- `self._global_model`: a freshly constructed model via `build_model(model_cfg)`.
- `self._global_model_bytes`: serialized bytes of the initial random model.
- `self._broadcast_epoch = 0`: tracks which epoch's model is currently available.
- `self._model_ready`: a `threading.Condition` used to signal waiting node threads when a new model is broadcast.
- Calls `_init_global_model()`.

---

#### `_init_global_model()` *(private)*

Serializes the randomly-initialized model immediately at startup. This becomes the epoch-0 model that all nodes fetch for their first round. Logs `GLOBAL_MODEL_INIT` with parameter count, byte size, and serde time.

---

#### `update_global_model(state_dict, epoch)`

Called by the aggregation thread after `FedAvg` completes.

1. Acquires `_model_lock`, calls `model.load_state_dict(state_dict)`, and re-serializes to update `_global_model_bytes` and `_broadcast_epoch`.
2. Acquires `_model_ready` and calls `notify_all()` to wake up any node threads blocked in `FetchGlobalModel`.

---

#### `Register(request, context)` ŌåÆ `RegistrationResponse`

Handles `Register` RPC.
1. Delegates to `registry.register(...)`.
2. Logs `NODE_REGISTERED`.
3. Returns a `RegistrationResponse` with `accepted=True`, the assigned partition index, and confirmation of the node ID.

---

#### `FetchGlobalModel(request, context)` ŌåÆ generator of `ModelChunk`

Handles the server-streaming `FetchGlobalModel` RPC. This is a Python generator function (`yield` inside).

1. **Wait for epoch**: Acquires `_model_ready` and waits in a loop until `_broadcast_epoch >= requested_epoch`. Checks `context.is_active()` every 5 seconds so a disconnected client doesn't leave a thread stuck forever.
2. **Privacy transform**: Calls `apply_privacy_transform(raw_payload)`.
3. **Chunk and stream**: Divides the payload into `ceil(total_bytes / chunk_size)` chunks. Yields each chunk as a `ModelChunk` proto message. Checks `context.is_active()` between chunks.
4. Logs `FETCH_MODEL_START` and `FETCH_MODEL_END` with byte count and streaming wall-time.

---

#### `SendLocalUpdate(request_iterator, context)` ŌåÆ `Ack`

Handles the client-streaming `SendLocalUpdate` RPC.

1. **Receive and reassemble**: Iterates the incoming chunk stream. On the first chunk, extracts `node_id`, `total_chunks`, `expected_bytes`, and logs `UPDATE_RECV_START`. Appends each chunk's bytes to a list.
2. **Join**: `raw_payload = b"".join(chunks)`.
3. **Privacy transform**: Calls `reverse_privacy_transform(raw_payload)`.
4. **Submit to registry**: Extracts `epoch` and `num_samples` from the last chunk's metadata. Calls `registry.submit_update(...)`.
5. Logs `UPDATE_RECV_END` and `UPDATE_SUBMITTED`.
6. Returns `Ack(success=accepted, ...)`.

---

### `run_aggregation_loop(servicer, registry, aggregator, logger, num_rounds, min_nodes)`

Runs in a dedicated non-daemon thread named `"aggregation-loop"`. The main coordinator loop.

For each epoch from 0 to `num_rounds - 1`:

1. Logs `ROUND_START`.
2. Calls `registry.wait_for_updates()` ŌĆö **blocks here** until K updates arrive or timeout fires.
3. If no updates arrived (complete timeout), logs `ROUND_TIMEOUT_NO_UPDATES` and advances the epoch.
4. Otherwise logs `ROUND_UPDATES_READY`, runs FedAvg inside a `TimedBlock("FEDAVG")`.
5. Calls `servicer.update_global_model(new_state, epoch + 1)` to broadcast.
6. Calls `registry.advance_epoch()` to reset the update collection.
7. Logs `ROUND_END`.

After all rounds: logs `ALL_ROUNDS_COMPLETE`.

---

### `serve(cfg, model_cfg)` ŌåÆ None

The top-level entry point called by `run_coordinator.py`.

1. Calls `ensure_log_dir(cfg)`.
2. Creates and starts `AsyncTelemetryLogger`.
3. Creates `NodeRegistry` and `FedAvgAggregator`.
4. Creates `FederatedLearningServicer`.
5. Creates `TelemetryServerInterceptor`.
6. Constructs the gRPC server with a thread pool of size `total_nodes + 4`, attaches the interceptor, registers the servicer, binds the port, and calls `server.start()`.
7. Launches `run_aggregation_loop` in a non-daemon thread.
8. Joins that thread (blocks until all rounds complete).
9. Calls `server.stop(grace=10)` ŌĆö waits up to 10 seconds for in-flight RPCs to finish.
10. Calls `logger.stop()` ŌĆö flushes remaining telemetry.

---

## 12. `fedsys/node/state_machine.py`

A strict finite-state machine that ensures the node's execution phases are always traversed in a valid order and that every phase transition is timestamped.

---

### `NodeState` (Enum)

Eight named states:

| State | Meaning |
|---|---|
| `IDLE` | Initial state before anything happens |
| `REGISTERING` | gRPC `Register` call in progress |
| `WAITING_FOR_MODEL` | Registered; waiting for coordinator to broadcast the epoch's model |
| `RECEIVING_MODEL` | `FetchGlobalModel` stream being consumed |
| `LOCAL_TRAINING` | Training loop running on local data |
| `TRANSMITTING_UPDATE` | `SendLocalUpdate` stream being sent |
| `DONE` | All rounds complete; terminal state |
| `ERROR` | Unrecoverable exception; terminal state |

---

### `_TRANSITIONS` (module-level dict)

Maps each `NodeState` to the `set` of states it is allowed to transition into:

```
IDLE               ŌåÆ {REGISTERING}
REGISTERING        ŌåÆ {WAITING_FOR_MODEL, ERROR}
WAITING_FOR_MODEL  ŌåÆ {RECEIVING_MODEL, DONE, ERROR}
RECEIVING_MODEL    ŌåÆ {LOCAL_TRAINING, ERROR}
LOCAL_TRAINING     ŌåÆ {TRANSMITTING_UPDATE, ERROR}
TRANSMITTING_UPDATEŌåÆ {WAITING_FOR_MODEL, DONE, ERROR}
DONE               ŌåÆ {}   (terminal)
ERROR              ŌåÆ {}   (terminal)
```

`WAITING_FOR_MODEL ŌåÆ DONE` covers the case where all rounds finish and the node exits the loop while still technically "waiting" for a model it will never need.

---

### `StateMachine`

#### `__init__(node_id, logger)`

Sets `self._state = NodeState.IDLE`, records `self._entered_at = time.perf_counter()`, and logs `STATE_ENTER` for IDLE immediately.

#### `state` (property) ŌåÆ `NodeState`

Read-only access to the current state.

---

#### `transition(new_state, **metadata)` ŌåÆ None

The core method. Called whenever the node changes phase.

1. Looks up `_TRANSITIONS[self._state]`. If `new_state` is not in the allowed set, raises `ValueError` with a clear message showing current state, attempted target, and allowed targets.
2. Computes `duration_ms` = time spent in the current state.
3. Logs `STATE_EXIT` with current state name, duration, and any extra metadata.
4. Updates `self._state = new_state` and `self._entered_at = now`.
5. Logs `STATE_ENTER` with the new state name.
6. If a callback was registered via `on_enter()` for this state, calls it.

---

#### `on_enter(state)` ŌåÆ decorator

Registers a zero-argument callback to be called whenever `state` is entered. Returns a decorator so it can be used as `@sm.on_enter(NodeState.LOCAL_TRAINING)`.

---

#### `is_terminal()` ŌåÆ `bool`

Returns `True` if the current state is `DONE` or `ERROR`.

---

## 13. `fedsys/node/trainer.py`

The computation layer. Contains everything that touches a GPU or CPU tensor. **Zero gRPC imports.** This is the file you modify to swap in Triton kernels or custom CUDA extensions.

---

### `run_local_training(model, global_state_bytes, dataloader, cfg, logger, epoch, num_samples=None)` ŌåÆ `(bytes, int)`

The single public function. Called by `FederatedNode._train_locally()` as a black box.

**Step 1 ŌĆö Determine device:**
Resolves `cfg.device` to a `torch.device`. Falls back to `"cpu"` if CUDA is requested but not available.

**Step 2 ŌĆö H2D transfer (inside `TimedBlock("H2D_TRANSFER")`):**
1. Calls `_load_state_dict(global_state_bytes, device)` to deserialize and map weights to the target device.
2. Calls `model.load_state_dict(state_dict)` to install the global weights.
3. Calls `model.to(device)` to ensure all buffers are on device.
4. Calls `model.train()` to enable dropout/batchnorm training mode.

If CUDA, resets `torch.cuda.reset_peak_memory_stats()` to get a clean VRAM measurement for this round.

**Step 3 ŌĆö Determine sample count:**
`n_samples = num_samples or len(dataloader.dataset)`.

**Step 4 ŌĆö Create optimizer and loss:**
- `Adam(model.parameters(), lr=cfg.learning_rate)`
- `BCEWithLogitsLoss()` ŌĆö combines sigmoid + binary cross entropy in one numerically stable operation.

**Step 5 ŌĆö Local epoch loop:**
For `local_epoch in range(cfg.local_epochs)`:
1. Calls `_train_one_epoch(...)` and gets the mean loss.
2. Logs `LOCAL_EPOCH_END` with the round index, local epoch index, and loss value.

**Step 6 ŌĆö Log training summary:**
Logs `LOCAL_TRAINING_END` with total elapsed ms and peak VRAM usage.

**Step 7 ŌĆö D2H transfer (inside `TimedBlock("D2H_TRANSFER")`):**
1. `cpu_state = {k: v.cpu() for k, v in model.state_dict().items()}` ŌĆö move every tensor to CPU.
2. `torch.save(cpu_state, BytesIO())` ŌĆö serialize to bytes.

Logs `GRADIENT_SERIALIZED` with byte size.

**Returns:** `(serialized_bytes, n_samples)`.

---

### `_train_one_epoch(model, loader, optimizer, criterion, device)` ŌåÆ `float` *(private)*

Runs one complete pass over the DataLoader.

For each batch:
1. Moves `user_ids`, `item_ids`, `labels` to the target device.
2. `optimizer.zero_grad(set_to_none=True)` ŌĆö more efficient than zeroing (releases memory).
3. `logits = model(user_ids, item_ids)` ŌĆö forward pass.
4. `loss = criterion(logits.squeeze(-1), labels)` ŌĆö compute binary cross-entropy.
5. `loss.backward()` ŌĆö compute gradients.
6. `optimizer.step()` ŌĆö update parameters.
7. Accumulates batch loss.

Returns `total_loss / n_batches` (mean loss over the epoch).

---

### `_load_state_dict(payload, device)` ŌåÆ `StateDict` *(private)*

Wraps `payload` bytes in `BytesIO` and calls `torch.load(buf, map_location=device, weights_only=True)`. The `weights_only=True` flag is a security measure preventing execution of arbitrary embedded Python code.

---

### `_peak_vram_mb(device)` ŌåÆ `float` *(private)*

If the device is CUDA: returns `torch.cuda.max_memory_allocated(device) / (1024┬▓)` in MiB. Returns `0.0` on CPU or any CUDA error.

## 14. `fedsys/node/client.py`

The networking layer for a training node. Drives the state machine, calls the trainer, and manages all gRPC communication. **Zero tensor operations** ŌĆö all compute is delegated to `trainer.py`.

---

### `FederatedNode`

#### `__init__(cfg, model_cfg, dataloader, num_rounds=10)`

1. Stores config, model config, dataloader, and round count.
2. Calls `ensure_log_dir(cfg)`.
3. Creates and starts `AsyncTelemetryLogger`.
4. Creates `StateMachine` (starts in IDLE state).
5. Sets `self._model = None` (will be created after registration).

---

#### `run()` ŌåÆ None

The main execution method. Called once per node.

```
_connect_and_register()
for fl_round in range(num_rounds):
    global_bytes = _fetch_global_model(fl_round)
    updated_bytes, n_samples = _train_locally(global_bytes, fl_round)
    _send_update(updated_bytes, n_samples, fl_round)
sm.transition(DONE)
```

Wrapped in a `try/except` that catches any exception, transitions to `ERROR`, re-raises, and always calls `logger.stop()` in `finally`.

---

#### `_connect_and_register()` *(private)*

1. Transitions state machine: `IDLE ŌåÆ REGISTERING`.
2. Creates a `TelemetryClientInterceptor` and wraps the gRPC channel with it via `grpc.intercept_channel(...)`.
3. Constructs the gRPC channel to `cfg.coordinator_address` with `cfg.grpc_options` (unlimited message sizes + keepalive).
4. Creates the gRPC stub: `federated_pb2_grpc.FederatedLearningStub(channel)`.
5. Builds a `NodeInfo` proto with `node_id`, dataset size, and device metadata.
6. Calls `stub.Register(node_info)` ŌĆö blocks until the coordinator responds.
7. Updates `cfg.data_partition` and `cfg.num_partitions` from the response.
8. Creates `self._model = build_model(self._model_cfg)`.
9. Transitions: `REGISTERING ŌåÆ WAITING_FOR_MODEL`.

---

#### `_fetch_global_model(epoch)` ŌåÆ `bytes` *(private)*

1. Transitions: `WAITING_FOR_MODEL ŌåÆ RECEIVING_MODEL`.
2. Sends `EpochRequest(node_id, epoch)` to `FetchGlobalModel`.
3. Iterates the server-side stream, collecting each `chunk.payload` into a list.
4. Joins: `raw_payload = b"".join(chunks)`.
5. Calls `reverse_privacy_transform(raw_payload)` to undo any transform applied by the coordinator.
6. Logs `FETCH_MODEL_DONE` with byte count, chunk count, network time, and transform time.
7. Transitions: `RECEIVING_MODEL ŌåÆ LOCAL_TRAINING`.
8. Returns the reassembled bytes.

---

#### `_train_locally(global_bytes, epoch)` ŌåÆ `(bytes, int)` *(private)*

1. Delegates entirely to `run_local_training(self._model, global_bytes, self._dataloader, cfg, logger, epoch)`.
2. Transitions: `LOCAL_TRAINING ŌåÆ TRANSMITTING_UPDATE`.
3. Returns `(updated_bytes, n_samples)`.

---

#### `_send_update(updated_bytes, n_samples, epoch)` *(private)*

1. Calls `apply_privacy_transform(updated_bytes)` ŌĆö outbound transform before chunking.
2. Computes `total_chunks = ceil(total_bytes / chunk_size)`.
3. Generates a UUID `epoch_id` for this specific upload.
4. Logs `SEND_UPDATE_START`.
5. Calls `stub.SendLocalUpdate(self._chunk_generator(...))` ŌĆö blocks until the server sends an `Ack`.
6. Logs `SEND_UPDATE_DONE` with `ack.success` and network time.
7. If `ack.success == False`, raises `RuntimeError` (rejected by server ŌĆö stale epoch or unknown node).
8. Transitions: `TRANSMITTING_UPDATE ŌåÆ WAITING_FOR_MODEL`.

---

#### `_chunk_generator(payload, total_chunks, chunk_size, epoch_id, epoch, n_samples)` ŌåÆ generator *(private)*

A Python generator that yields one `ModelChunk` proto per iteration.

For each chunk index `i`:
```
start = i * chunk_size
end   = min(start + chunk_size, total_bytes)
yield ModelChunk(
    payload      = payload[start:end],
    chunk_index  = i,
    total_chunks = total_chunks,
    epoch_id     = epoch_id,
    total_bytes  = total_bytes,
    metadata     = {"node_id": ..., "epoch": ..., "num_samples": ...},
)
```

The server reads these chunks from `SendLocalUpdate`'s `request_iterator`. The metadata dict on each chunk lets the server know which node sent it and at which epoch, without needing extra RPCs.

## 15. `proto/federated.proto`

Defines the gRPC service contract. This is the single source of truth for the wire format. Both the Python stubs in `fedsys/generated/` are auto-compiled from this file.

### Service: `FederatedLearning`

```protobuf
service FederatedLearning {
    rpc Register         (NodeInfo)            returns (RegistrationResponse);
    rpc FetchGlobalModel (EpochRequest)        returns (stream ModelChunk);
    rpc SendLocalUpdate  (stream ModelChunk)   returns (Ack);
}
```

| RPC | Request | Response | Pattern |
|---|---|---|---|
| `Register` | `NodeInfo` | `RegistrationResponse` | Unary (one request, one response) |
| `FetchGlobalModel` | `EpochRequest` | `stream ModelChunk` | Server-streaming (one request, many response chunks) |
| `SendLocalUpdate` | `stream ModelChunk` | `Ack` | Client-streaming (many request chunks, one response) |

### Messages

**`NodeInfo`** ŌĆö sent by a node during registration:
- `node_id` (string): unique identifier
- `address` (string): node's network address (informational)
- `local_dataset_size` (int32): how many training samples this node has
- `metadata` (map): free-form string key-value pairs

**`RegistrationResponse`** ŌĆö coordinator's reply to `Register`:
- `accepted` (bool): whether registration was accepted
- `message` (string): human-readable status
- `assigned_partition` (int32): which data shard to load
- `total_partitions` (int32): total number of shards
- `confirmed_node_id` (string): echoes back the node_id

**`EpochRequest`** ŌĆö sent by a node to request the current global model:
- `node_id` (string)
- `epoch` (int32): the epoch the node wants

**`ModelChunk`** ŌĆö one binary chunk of a model payload:
- `payload` (bytes): raw opaque bytes
- `chunk_index` (int32): position in the sequence
- `total_chunks` (int32): total number of chunks in this transfer
- `epoch_id` (string): UUID identifying this specific transfer
- `total_bytes` (int64): full payload size (for validation)
- `metadata` (map): carries `node_id`, `epoch`, `num_samples` on upload chunks

**`Ack`** ŌĆö coordinator's reply to `SendLocalUpdate`:
- `success` (bool)
- `message` (string)
- `node_id` (string): echoes back who sent it
- `epoch` (int32): echoes back the epoch

## 17. `scripts/generate_synthetic_data.py`

### `parse_args()` ŌåÆ `Namespace`

Defines command-line arguments:
- `--num-users`: how many unique users (default 1000)
- `--num-items`: how many unique items (default 500)
- `--num-samples`: total interactions across all partitions (default 10000)
- `--num-partitions`: how many CSV shard files to produce (default 2)
- `--latent-dim`: dimensionality of the hidden preference/feature vectors (default 16)
- `--seed`: random seed for reproducibility (default 42)
- `--out-dir`: output directory (default `data/synthetic/`)

### `dot(a, b)` ŌåÆ `float`

Standard dot product of two Python lists.

### `sigmoid(x)` ŌåÆ `float`

Numerically stable sigmoid: `1 / (1 + exp(-x))`.

### `randn(rng, dim)` ŌåÆ `list[float]`

Generates `dim` normally-distributed values using the Box-Muller transform applied to a `random.Random` instance. This avoids a NumPy dependency in the generation script.

### `main()` ŌåÆ None

The generation pipeline:

1. Seeds `random.Random(args.seed)`.
2. **Latent vectors**: Samples `user_vecs[u] ~ N(0,I)` for all users and `item_vecs[i] ~ N(0,I)` for all items, dimension `latent_dim`.
3. **Generate interactions**: For each of `num_samples` samples:
   - Draws random `(u, i)` pair.
   - Computes `score = dot(user_vecs[u], item_vecs[i]) / sqrt(D)`.
   - Probability `p = sigmoid(score)`.
   - Label = `Bernoulli(p)` ŌĆö biased by the user's true preference for the item.
   - Assigns to partition `u % num_partitions`.
4. **Write CSVs**: Shuffles each partition and writes `partition_0.csv`, `partition_1.csv`, etc.
5. **Write meta.json**: Records all generation parameters for the dataset loader to read back.

The latent-factor structure means users with similar preference vectors to an item will tend to have positive interactions ŌĆö the model can actually learn to predict this.

## 19. `scripts/run_node.py`

### `parse_args()` ŌåÆ `Namespace`

Argument groups:
- Node identity: `--node-id`, `--coordinator`
- Training: `--device`, `--local-epochs`, `--batch-size`, `--lr`, `--num-rounds`
- Data source (mutually exclusive group):
  - `--data-dir DIR`: load pre-generated CSV files (recommended)
  - `--amazon`: download the real Amazon 2023 dataset
  - *(neither)* ŌåÆ in-memory synthetic (default, fastest)

- Partition: `--num-partitions`, `--partition`

### `main()` ŌåÆ None

1. Parses args, splits `--coordinator` into host and port.
2. Generates `node_id` from UUID if not provided.
3. Builds `NodeConfig`.
4. Builds `ModelConfig` from `--model-type` where valid choices are:
   - `simple`
   - `bpr`
   - `neural_cf`
   - `two_tower`
5. **Data source routing**:
   - `--data-dir`: calls `load_meta(args.data_dir)`, overrides `model_cfg.num_users` and `num_items` from meta, calls `build_synthetic_dataloader(...)`.
   - `--movielens`: loads MovieLens, partitions users, builds BPR-style pair dataloader.
   - default: calls in-memory synthetic fallback.
6. Creates `FederatedNode(cfg, model_cfg, dataloader, num_rounds)`.
7. Calls `node.run()` ŌĆö blocks until all rounds complete.
